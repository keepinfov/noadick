# noadick

Игровой Telegram-бот на [aiogram 3](https://docs.aiogram.dev): ежедневная игра
«измерь писюн», дуэли между игроками, топ чата и система «болезней» (баффы/дебаффы).
Состояние хранится в SQLite. Есть глобальная админ-панель прямо в Telegram.

## Команды

| Команда | Описание |
|---------|----------|
| `/dick` | раз в день меняет размер на случайную величину |
| `/duel [ставка]` | вызвать на дуэль (ответом на сообщение или открытый вызов) |
| `/top` | топ-10 игроков чата |
| `/help` | список команд |
| `/ping` | проверка отклика |
| `/admin` | админ-панель (только для ID из `ADMIN_IDS`) |

## Админ-панель (`/admin`)

Доступна пользователям, чьи Telegram ID перечислены в переменной `ADMIN_IDS`.
Инлайн-меню позволяет:

- просматривать список всех чатов и игроков в них;
- менять размер игрока (±1/±10, задать точное значение), переименовывать;
- выдавать/снимать болезни, сбрасывать и удалять игроков;
- сбрасывать весь чат;
- банить пользователей и чаты;
- искать игрока по ID или имени во всех чатах;
- смотреть глобальную статистику;
- делать рассылку по всем чатам.

Все операции изменения вынесены в `services/admin_actions.py` (без зависимости от
aiogram) и пишут лог в таблицу `audit_log`, поэтому их можно переиспользовать в
будущей веб-панели.

## Конфигурация

Скопируй `.env.example` в `.env` и заполни:

```
BOT_TOKEN=...        # токен от @BotFather (обязательно)
ADMIN_IDS=111,222    # Telegram ID админов через запятую
DB_PATH=./data/bot.db
TZ=Europe/Moscow
PROXY=               # опциональный socks5/http прокси
```

## Запуск

### Docker (рекомендуется)

```bash
docker compose up --build -d
```

База лежит в `./data/bot.db` (том примонтирован, переживает рестарт).

### Локально

```bash
python -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
python bot.py
```

## Миграция старых данных

Раньше состояние хранилось в `storage/<md5(chat_id)>.json`. Имя файла —
необратимый md5, поэтому реальный `chat_id` неизвестен. Скрипт миграции загружает
файлы в таблицу-стейджинг `legacy_chats` по их хешу:

```bash
python -m scripts.migrate_json ./storage
```

При первом сообщении из чата бот вычисляет `md5(chat_id)`, находит совпадение и
автоматически привязывает старые данные к реальному `chat_id`
(`services/registry.py`). До прихода сообщения такой чат в админке не виден — это
неизбежное следствие необратимости md5.

## Архитектура

```
bot.py                  — точка входа: init_db, middleware, роутеры
db/                     — SQLAlchemy async движок и модели (Chat, User, Player, AuditLog, LegacyChat)
repositories/           — доступ к данным (players, chats)
services/               — переиспользуемая логика: admin_actions, registry, game
middlewares/registry.py — регистрация чатов/юзеров, авто-привязка legacy, баны
handlers/               — команды бота, включая admin.py
models/disease.py       — система болезней
scripts/migrate_json.py — импорт старых JSON
```
